# Implementation Plan: Fix DC Plan Match/Core Contributions When Disabled

**Branch**: `069-fix-match-core-disabled` | **Date**: 2026-03-13 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/069-fix-match-core-disabled/spec.md`

## Summary

The DC Plan UI's "Match Enabled" toggle sends `match_enabled: false` to the API, but the orchestrator config export ignores this field and the dbt match calculation model has no master on/off gate. Match contributions are always calculated regardless of the toggle state. The fix adds `employer_match_enabled` variable propagation through the config export pipeline and a master gate in the dbt model, following the proven `employer_core_enabled` pattern. Core contributions are already correctly wired — no core fix needed.
