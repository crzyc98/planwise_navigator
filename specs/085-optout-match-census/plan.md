# Implementation Plan: Match Census for Opt-Out Rate Configuration

**Branch**: `085-optout-match-census` | **Date**: 2026-05-20 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/085-optout-match-census/spec.md`

## Summary

Add a "Match Census" button to the Opt-Out Assumptions panel in PlanAlign Studio. When clicked, the system analyzes the uploaded census file to calculate the non-participant rate among employees hired within a configurable tenure lookback window (default 3 years), then presents a preview for the analyst to review and optionally apply as the target opt-out rate. Implementation follows the established `TurnoverSection.tsx` match-census pattern on the frontend and the `TurnoverAnalysisService` pattern on the backend. No dbt model changes or database schema changes are required.

## Technical Context

**Language/Version**: Python 3.11 (backend), TypeScript/React 18 (frontend)
**Primary Dependencies**: FastAPI + Pydantic v2 (backend), React 18 + Tailwind CSS v4 (frontend), DuckDB in-memory (census analysis)
**Storage**: No new storage; census analysis reads files in-memory; opt-out rate value persisted via existing scenario config JSON
**Testing**: pytest with `@pytest.mark.fast` for unit tests; FastAPI TestClient for integration tests
**Target Platform**: Linux server (backend), modern browser (frontend)
**Project Type**: Web service (FastAPI backend + React/Vite frontend)
**Performance Goals**: Census analysis completes in <3 seconds for files up to 100K rows
**Constraints**: No dbt model changes; no simulation database schema changes; SQL column names must pass security validation
**Scale/Scope**: Single new endpoint, one new service file, one new model file, two modified files (backend), two modified files (frontend)

### Phase B: Backend â€” Service

**Goal**: Implement the census analysis logic.

**Tasks**:

3. **Create `planalign_api/services/opt_out_service.py`**

   ```
   class OptOutAnalysisService:
       __init__(workspaces_root: Path)
       analyze_opt_out_rate(workspace_id, file_path, lookback_years) -> OptOutRateAnalysisResult
   ```

   **Service logic** (mirrors `TurnoverAnalysisService` structure):
   - Resolve and validate file path using `validate_file_path_for_sql()`
   - Load CSV/Parquet into DuckDB in-memory connection
   - Auto-detect hire_date column: iterate `CENSUS_HIRE_DATE_COLUMNS`, pick first match
   - Auto-detect deferral_rate column: iterate `CENSUS_DEFERRAL_COLUMNS`, pick first match
   - Raise `ValueError` if neither required column is found (with message listing expected names)
   - Count total eligible (active employees with non-null hire_date)
   - Count excluded_null_tenure (active employees with null hire_date)
   - Calculate lookback_cutoff = MAX(hire_date) - (lookback_years Ă— 365 days)
   - Count eligible_count = active employees WHERE hire_date >= lookback_cutoff
   - Count non_participant_count = eligible_count WHERE deferral_rate = 0 OR deferral_rate IS NULL
   - Compute suggested_rate = non_participant_count / eligible_count (None if eligible_count = 0)
   - Populate warning message if eligible_count < 20 (low-confidence window)
   - Return `OptOutRateAnalysisResult`

   **Active employee detection** (case-insensitive):
   ```sql
   UPPER(CAST(active AS VARCHAR)) IN ('ACTIVE', 'Y', '1', 'TRUE', 'YES')
   OR active IS NULL  -- treat NULL active as active (permissive fallback)
   ```
