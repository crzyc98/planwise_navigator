# Research: scheduled_hours_per_week + Part-Time Eligibility (#282)

## Decision: Nullable DECIMAL(5,2) column in staging
**Rationale**: Backward-compatible — NULL → 40 hrs/wk default preserves all existing behavior. No migration needed.
**Alternatives considered**: Non-nullable with default 40 — rejected because it would make census data appear to have a value that wasn't supplied.

## Decision: Replace 2080.0 with `COALESCE(scheduled_hours_per_week, 40.0) * 52.0`
**Rationale**: Pure formula swap. `40 * 52 = 2080` so NULL rows are unchanged. Avoids a new config variable or dbt variable.
**Alternatives considered**: dbt variable `{{ var('default_annual_hours', 2080) }}` — more flexible but adds complexity not needed now.

## Decision: Part-time new hires use deterministic hash (same pattern as age assignment)
**Rationale**: `int_hiring_events.sql` already uses `ABS(MOD(HASH(CONCAT(...)), 1000000.0)) / 1000000.0` for age bucketing. A second hash with `'_pt_'` salt gives independent deterministic assignment.
**Formula**:
```sql
ABS(MOD(HASH(CONCAT(CAST(hire_sequence_num AS VARCHAR), '_pt_', CAST({{ simulation_year }} AS VARCHAR)))::DOUBLE, 1000000.0)) / 1000000.0 < part_time_new_hire_pct
```

## Decision: Magic button endpoint follows `/analyze-age-distribution` pattern exactly
**Rationale**: Same request shape (`FileValidationRequest`), same service-layer delegation, same error handling. Minimal code surface.
**Return shape**:
```json
{ "part_time_pct": 0.18, "headcount": 500, "part_time_count": 90, "column_present": true }
```
Button is disabled when `column_present = false`.

## Decision: `scheduled_hours_per_week` carried forward in `int_workforce_previous_year.sql`
**Rationale**: Part-time status is persistent — an employee hired at 20 hrs/week stays part-time year-over-year unless a RAISE/PROMOTION event changes their schedule. Carrying the column forward avoids re-reading census each year.
**Alternatives considered**: Re-derive from census each year — rejected because new hires generated by simulation don't have a census row.
